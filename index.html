Chapter 2

When Everything Broke

The pager went off. The dashboard turned red. These four P0 incidents changed how we think about inference infrastructure.

1
KV Cache Eviction Storm
3x traffic spike at 02:47 AM. Cache hit rate cliff-dived from 85% to 3% in 90 seconds. Pure LRU evicted hot system prompts alongside one-off completions. SLO violated for 94% of requests over 35 minutes.
2
Model Download Thundering Herd
Rolling update triggered 24 pods pulling 140GB simultaneously. Saturated the 10Gbps link. Model load went from 4 min to 47 min. Complete outage for 38 minutes.
3
RDMA Network Partition
Switch failure at 04:12. TCP was fine so pods looked healthy. RDMA was dead. Prefill pods couldn't transfer KV cache to decode pods. Every dashboard showed green. Users saw 100% failure.
4
Prefill Pod Starvation
50 concurrent 128K-context requests at 09:15. Short-context TTFT jumped from 200ms to 45s -- a 225x degradation. One 128K request costs 256x more compute than a 500-token request. Same queue = guaranteed starvation.
The One Thing I Wish Someone Told Me
Cache eviction is a cliff, not a slope. By the time you see metrics degrade, the cascade has already started. Set admission control thresholds 15% below your eviction trigger point.
About Disaggregated Serving
You have TWO networks. Kubernetes only monitors one of them. The other one will fail silently at 4 AM and every dashboard will show green.
Chapter 3

The Silent Killers

Not every failure sets off alarms. These incidents degraded performance for hours -- even days -- before anyone noticed.

Autoscaler Oscillation
HPA thrashed 8 to 20 pods every 5 minutes. Default stabilization window too short. 30% of fleet perpetually loading models. 15% error rate from killed requests.
GPU Memory Fragmentation
OOM kills with 40% free memory. nvidia-smi lies -- it shows total free, not contiguous free. After 18 hours of mixed workloads, long-context requests fail silently.
NUMA Node Misplacement
4 of 12 pods at 38ms/tok instead of 18ms/tok. GPU looked fine. Memory path crossed NUMA boundaries. Took 3 days to find. Invisible to every dashboard.
Health Check False Positives
vLLM returns HTTP 200 before model weights are on GPU. Kubernetes believed it. 5,000 timeouts during rolling updates. "The new deploy is broken." It wasn't -- it was premature.
Token Budget Exhaustion
"The model got dumber." 340K silently truncated responses over 3 days. finish_reason=length on 40% of responses. Nobody monitored that field. Zero alerts fired.
Decode Latency from Defrag
Fixed fragmentation with a defrag job. New problem: decode latency spikes 5x every 6 hours during defrag. The fix needed its own fix.
Prefix Cache Collision
"The model is hallucinating." It wasn't -- hash collision in multi-tenant prefix cache. Tenant B got completions influenced by Tenant A's context. 4 days to identify.
ConfigMap Race Condition
ConfigMap update at 10:45. Half the fleet routed one way, half the other for 30-90 seconds. Cache hit rate dropped from 72% to 11%. Session affinity broken.
Chapter 4

Patterns We Learned

After 12 incidents, these lessons crystallized into principles that now guide every production deployment.

⚠
Cache Eviction IS Your SLO Policy
LRU is not enough for production inference. Frequency-weighted eviction keeps hot system prompts pinned. Add cache pressure metrics to EPP routing decisions and admission control to shed load before capacity is exhausted.
🔌
Model Weights Are Infrastructure
Treat model loading as a first-class infrastructure concern. Pre-pull to node-local NVMe via DaemonSet. Use maxSurge=1, maxUnavailable=0. Slow rollouts are correct rollouts.
📈
Not All Tokens Are Equal
A 128K prefill is 256x more expensive than a 500-token prefill. Deploy priority queues by context length. Without them, your shortest requests get hurt the most.
🔍
Monitor What Matters
nvidia-smi total free is not enough -- track fragmentation ratio. Monitor finish_reason distribution. Alert when truncation exceeds 5%. Silent truncation is worse than a loud error.
⚙
LLM Autoscaling Is Different
Autoscaling LLM pods is not like scaling web servers. Model load time means every scale event costs minutes, not seconds. Set HPA stabilization to 15+ minutes. Err on the side of keeping capacity.
🔒
Config Changes Are Rolling Updates
Eventually consistent ConfigMap propagation breaks session affinity for 30+ seconds. Add config version to health checks. Treat config changes as rolling updates, not instant switches.
